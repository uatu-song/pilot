# Montage Style Guide - Non-Linear Intercutting

**When to Use:** Training sequences, power development, team formation moments where you need to show cause (training) and effect (application) efficiently.

**When NOT to Use:** Default narrative style, emotional processing, mystery reveals, dialogue-heavy scenes, or any time linear flow serves the story better.

## Structure Pattern (Chapter 16 Example)

1. **Setup** - Establish present surveillance/observation point
2. **Flash Forward** - Show the climactic event beginning
3. **Flash Back** - Return to training/preparation ("But before they did...")
4. **Intercut Pattern** - Discovery → Immediate application at climactic event
5. **Repeat** - Same pattern for each character/power
6. **Resolve** - Full team synergy at climactic event, then aftermath

## Key Principles

### Always Orient the Reader
Use clear time markers:
- "A few nights later..."
- "But before they did, many days ago..."
- "At the community center..."
- Never leave the reader guessing WHEN they are

### Repeat the Pattern for Clarity
If Character A gets: Discovery → Field Test
Then Characters B, C, D get: Discovery → Field Test

Repetition creates understanding, not confusion.

### Build Toward ONE Specific Event
All timelines converge on a single moment. Don't randomly time-jump—every flashback/forward serves the central event.

### Form Reinforces Content
Use when structure enhances theme:
- Time manipulation powers = non-linear structure
- Memory/trauma = fragmented chronology
- Cause/effect relationships benefit from seeing effect first

### Limit Complexity
**Maximum two timelines** for clarity:
- Past (training/preparation)
- Present (application/climactic moment)

Don't add a third unless absolutely necessary.

## Chapter 16 Breakdown

**Timeline A (Present):**
- CADENS facility surveillance
- Community center raid (the event)
- Aftermath/consequences

**Timeline B (Past - Days Before):**
- Training at Parkour Academy
- Power trigger discoveries
- Individual testing

**Intercutting Pattern:**
Each character gets:
1. Discovery attempt at academy (Timeline B)
2. Ahdia/Ryu assign trigger (Timeline B)
3. Immediate cut to using that trigger at raid (Timeline A)
4. Physical cost to Ahdia (Timeline A)
5. Next character

**Result:** Reader sees training and application simultaneously, building toward team synergy moment.

## Information Delivery in Montage

Use this structure to deliver multiple types of info efficiently:

**Through Action:**
- Powers being discovered and used
- Team dynamics forming
- Physical consequences escalating

**Through Dialogue:**
- Mechanism explanations (Ryu's technical breakdown)
- Character reactions/realizations

**Through Visuals:**
- Biometric data screens
- Social media feeds
- Surveillance footage

**Through Subtext:**
- Ryu documenting rather than intervening
- Firas's absence from celebrations
- Ahdia's deterioration curve

## Pacing Within Montage

**Each segment should:**
- Be brief (1-3 paragraphs per discovery)
- Maintain rhythm (same beats for each character)
- Build cost progressively (Ahdia's symptoms worsen)
- Escalate toward climax (individual → team synergy)

**Avoid:**
- Extended dialogue mid-montage (breaks rhythm)
- Overexplaining (trust the pattern)
- Varying structure per character (creates confusion)

## When This Structure Works Best

✓ Training → application sequences
✓ Power/skill development arcs
✓ Team coordination building to synchronized moment
✓ Showing methodology before showing result
✓ Time/memory thematically relevant scenes
✓ Information-dense chapters that need momentum

## When to Use Linear Instead

✓ Emotional processing/character revelations
✓ Mystery/suspense requiring information control
✓ Immediate action tension (non-linear deflates urgency)
✓ Dialogue-heavy exposition
✓ Default narrative flow
✓ Any scene where chronology matters to understanding

## Key Takeaway

**Montage style is a tool, not the default.** Use it when:
1. You need efficiency (show cause and effect simultaneously)
2. Form enhances content (structure serves theme)
3. Pattern creates clarity (repetition aids understanding)
4. Multiple information types need delivery (action + dialogue + visual + subtext)

Otherwise, stick to linear narrative flow.
